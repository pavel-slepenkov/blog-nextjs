Setting up web scraping python environment on Google Compute Engine

December 10, 2018

Disclaimer : Nothing unique is described here. Post is made with intent to have a checklist of my setup. It’d be great if it might be interested or helpful to someone else

One of my project requires a daily web scraping and storing data on database. Project includes the following components:

  • Python 3.7
  • Postgres
  • Cron

Past year this project was running on AWS EC2 t2.micro instance and this was enough. I’m going to switch AWS to GCP in order to test new service

**1. Setup [Google Cloud Platform ](http://cloud.google.com)account** It’s super easy to fill in several forms and get your account.

2. Setup Debian machine. I choose g1-small machine with 1 vCPU and 1.7 GB memory which should be enough for my experiments. During machine configuration process you can add your ssh key.

3. Connect via SSH. After setup you can login with SSH directly from browser or by your preferable client with ssh key provided in the previous step

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: bash; notranslate&quot;&gt; ssh -i &quot;~/.ssh/id_rsa&quot; pavel@32.248.5.168

where pavel is name which I give to my ssh key and 32.248.5.168 is public IP address of my instance.

After login to the newly created instance I have to setup all required tools

4. Python installation Debian machine comes with Python 3.5 installed. My projects use python 3.7, so I need to install this Python version.

Install build-essential

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: bash; notranslate&quot;&gt;sudo apt-get install build-essential

Build Python from sources

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: bash; notranslate&quot;&gt;wget https://www.python.org/ftp/python/3.7.1/Python-3.7.1.tgz
tar xvf Python-3.7.1.tgz
cd Python-3.7.1
./configure --enable-optimizations
make -j8
sudo make altinstall
python3.7

It took about 40 minutes

**5. Postgres installation** Official postgres site provides [great instruction](https://www.postgresql.org/download/linux/debian/) on how to install postgres version which you need. create file 
&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: plain; notranslate&quot;&gt;sudo vi /etc/apt/sources.list.d/pgdg.list

with the following content

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: plain; notranslate&quot;&gt;deb http://apt.postgresql.org/pub/repos/apt/ stretch-pgdg main

Then run

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: bash; notranslate&quot;&gt;wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -
sudo apt-get update
apt-get install postgresql-11

After success Postgres installation we need to perform initial configuration. Setup postgres user password by the following command.

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: bash; notranslate&quot;&gt;sudo -u postgres psql -c &quot;ALTER USER postgres PASSWORD &#39;your_password&#39;;&quot;
sudo vim /etc/postgresql/11/main/pg_hba.conf

change config file with the following values

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: plain; notranslate&quot;&gt;# Database administrative login by Unix domain socket
local   all             postgres                                md5

# &quot;local&quot; is for Unix domain socket connections only
local   all             all                                     md5

Restart Postgres

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: plain; notranslate&quot;&gt;sudo /etc/init.d/postgresql restart

Create new user for using in your development environment

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: plain; notranslate&quot;&gt;createuser -U postgres -d -e -E -l -P -r -s &lt;my_name&gt;

6. Configure crontab run crontab -e and add the following lines to its config

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: bash; notranslate&quot;&gt;SHELL=/bin/bash
PYTHONIOENCODING=utf8
0 0 15 * * python /home/pavel/projects/p1/src/scraper1.py all
0 * * * * python /home/pavel/projects/p1/src/scraper1.py latest
0 * * * * python /home/pavel/projects/p1/src/s1/scraper2.py

7. Create a swap file Google Cloud Compute vm had no swap file. So we need to create one

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: bash; notranslate&quot;&gt;sudo dd if=/dev/zero of=/var/swap bs=2048 count=524288
sudo chmod 600 /var/swap
sudo mkswap /var/swap
sudo swapon /var/swap

And then the following line to /etc/fstab to make it permanent.

&lt;pre class=&quot;wp-block-syntaxhighlighter-code brush: plain; notranslate&quot;&gt;/var/swap none swap sw 0 0
