BULK remove of DatedConversionRate

June 7, 2019

I screw up DatedConversionRate data loading with ~2800 records created with incorrect StartDate. As I can retrieve it via SOQL I quickly created some apex script. Something like that

&lt;pre class=&quot;wp-block-syntaxhighlighter-code&quot;&gt;delete [
    SELECT Id
    FROM DatedConversionRate
    WHERE CreatedDate = TODAY
    AND StartDate &lt; 2018-01-01
];

and got this

&lt;pre class=&quot;wp-block-syntaxhighlighter-code&quot;&gt;DML operation Delete not allowed on List&lt;DatedConversionRate&gt;

Not cool at all. Also I could not update StartDate with error

&lt;pre class=&quot;wp-block-syntaxhighlighter-code&quot;&gt;DML operation Update not allowed on List&lt;DatedConversionRate&gt;

Okay. Quick googling give me the solution. Here’s solution for create/update and delete DatedConversionRate via SOAP API. So, basically you can do it from Apex

&lt;pre class=&quot;wp-block-syntaxhighlighter-code&quot;&gt;HttpRequest req = new HttpRequest();
req.setEndpoint(URL.getSalesforceBaseUrl().toExternalForm() + &#39;/services/data/v45.0/sobjects/DatedConversionRate/04w0e0000000Po1AAE?_HttpMethod=DELETE&#39;);
req.setHeader(&#39;Authorization&#39;, &#39;OAuth &#39; + UserInfo.getSessionId());
req.setHeader(&#39;Content-Type&#39;, &#39;application/json&#39;);
req.setMethod(&#39;POST&#39;);
HttpResponse res = (new Http()).send(req);

But because I had ~2800 records to remove I decided to go with python

&lt;pre class=&quot;wp-block-syntaxhighlighter-code&quot;&gt;import request

record_ids_to_delete = [&#39;04w0e0000000Po1AAE&#39;,&#39;04w0e0000000Po0AAE&#39;,&#39;04w0e0000000PnzAAE&#39;, ...2800 ids...]
headers = {&quot;Authorization&quot;: &quot;Bearer AQMAQPJ.ZAiHxG.LUaDqgUabvVdCao77Lq2R2kzcOP0fwMk7lbHA4G7va4U1jRfL_W0hFRBswLYTox2q1yLAsfL93MismKCC&quot;, &quot;Content-Type&quot;: &quot;application/json&quot;}
for record_id in record_ids_to_delete:
    requests.post(&#39;https://my-org-name.my.salesforce.com/services/data/v45.0/sobjects/DatedConversionRate/&#39; + record_id + &#39;?_HttpMethod=DELETE&#39;, headers=headers)

this script might be much more improved with session id generation and using SOQL for retrieving record ids. But I hope that this is one time thing and copying/pasting session id from browser/any other source of session id as well as quick generation of monstrous array in sublime text are not so big deal when you have to fix it asap.
